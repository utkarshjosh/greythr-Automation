# Flutter App Design Guidelines

## Design Principles

### 1. Material Design 3

- Use Material Design 3 components and patterns
- Follow Material You color system
- Implement dynamic color theming
- Use Material 3 typography scale

### 2. Visual Hierarchy

- Clear information architecture
- Consistent spacing (8px grid system)
- Visual weight through size, color, and contrast
- Progressive disclosure of information

### 3. User Experience

- Smooth animations and transitions (200-300ms)
- Clear feedback for all user actions
- Loading states and error handling
- Pull-to-refresh where appropriate
- Empty states with helpful messages

### 4. Color System

- Primary: Indigo (#6366F1)
- Secondary: Purple (#8B5CF6)
- Accent: Cyan (#06B6D4)
- Status Colors:
  - Success/Done: Green (#10B981)
  - Pending: Amber (#F59E0B)
  - Error: Red (#EF4444)
  - Skip: Gray (#6B7280)

### 5. Typography

- Display Large: 32px, Bold
- Display Medium: 28px, Bold
- Headline: 20px, Semi-bold
- Title: 18px, Semi-bold
- Body: 16px, Regular
- Label: 14px, Medium
- Caption: 12px, Regular

### 6. Spacing

- Use 8px base unit
- Common spacing: 8, 16, 24, 32px
- Card padding: 20px
- Section spacing: 24px

### 7. Components

- Cards: 16px border radius, elevation 2
- Buttons: 12px border radius, 16px vertical padding
- Inputs: 12px border radius, filled style
- Badges: Rounded with icon + text

### 8. Animations

- Page transitions: 300ms
- Card interactions: 200ms
- Loading indicators: Smooth rotation
- Status changes: Fade + scale

### 9. Accessibility

- Minimum touch target: 48x48px
- Color contrast: WCAG AA compliant
- Semantic labels for screen readers
- Support for system font scaling

### 10. Responsive Design

- Support different screen sizes
- Adaptive layouts for tablets
- Safe area handling
- Landscape orientation support
